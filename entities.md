## Users

Описание: хранит информацию о зарегистрированных пользователях, их учетные данные, профиль и статус присутствия в системе.

### Поля

| Поле          | Тип                               | Обязательное | Описание                                                    |
|---------------|-----------------------------------|--------------|-------------------------------------------------------------|
| id            | INTEGER                           | Да           | Уникальный идентификатор (PK)                               |
| global_role   | enum (user, moderator)            | Да           | Роль пользовтеля                                            |
| username      | VARCHAR(30)                       | Да           | Ник пользователя, до 30 символов                            |
| email         | VARCHAR(100)                      | Да           | Почта                                                       |
| password_hash | VARCHAR(255)                      | Да           | Хеш пароля пользователя                                     |
| bio           | VARCHAR(100)                      | Нет          | Описание профиля                                            |
| last_seen     | TIMESTAMP                         | Да           | Дата и время последней активности, default: created_at      |
| is_online     | BOOLEAN                           | Да           | Статус онлайн/оффлайн, default: false                       |
| created_at    | TIMESTAMP                         | Да           | Дата создания                                               |
| updated_at    | TIMESTAMP                         | Да           | Дата последнего обновления                                  |

Если не указать WHERE то изменения затронут все записи 

Что такое первичный ключ (PRIMARY KEY)? Зачем он нужен?

Уникальный ключ по которому можно находить нужную строку в таблице. Еще на неё ссылаются FK
